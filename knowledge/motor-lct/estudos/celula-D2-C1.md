## 🧠 Síntese (knowledge-synthesis)

### O que une as 3 fontes

| Elemento | LUZ HUB | Motor (C1) | The Revenue Codex |
|---|---|---|---|
| **Tese central** | Accountability trimestral | Métricas 20/80 + CAC/LTV | Trio CAC+Payback+LTV explicita 80% da saúde |
| **Método** | 2 sessões/mês + WhatsApp | Dashboard trimestral | 6 KPIs North-Star |
| **Mecanismo** | Vínculo CEO↔Leandro via números | Cobrança vs. ausência | North-Star Dashboard 1 página |
| **Saída** | D3 (Diagnóstico) acionado por gaps no Dashboard | Plano de ação quando 1 KPI vermelhidar | Painel imprimido numa A4 |

### Insights cruzados (achados do estudo)

1. **D2×C1 é a "calculadora de relacionamento"** — a LCT não vende mentoria como informação, vende como **accountability**. O cliente fica porque vê os números e só consegue justificar/festejar resultados com base neles.

2. **O trio CAC-Payback-LTV é o teste ácido LCT** — não o trio NPS-Csat-CES (vendidos por consultorias mais superficiais). **Por quê?** Porque o ICP B2B tech precisa de números que expliquem para o **board**, não só para si mesmo.

3. **Churn de receita + NRR é a lente LCT para retenção** — outras consultorias veem "satisfação do cliente" (Cap. 25 do The Revenue Codex). LCT vê **expansão líquida da base atual** (NRR). **Diferença crucial para o ICP B2B tech**.

4. **O North-Star Dashboard do Cap. 32 (6 KPIs) é o formato exato do entregável** — MRR, CAC, Payback, NRR, Churn, Leads qualificados. **D2×C1 não inventa; importa o padrão do Compêndio.**

5. **Ciclo trimestral (não mensal)** — alinhado com o LUZ HUB §"Accountability trimestral contra KPIs acordados". Cadência LCT = leitura mensal informal via WhatsApp + reunião trimestral formal com o dashboard impresso.

6. **Atenção aos choques:** C5 (Métricas SaaS vs métricas de teto/saída) **se aplica parcialmente** aqui. O LCT pode adaptar as 6 KPIs para cliente ICP. Por exemplo: ISP não usa MRR puro; usa "Monthly Recurring Telecom Revenue". **Ajuste cosmético no nome, sem perder o conteúdo.**
